# RentEase - Система аренды недвижимости

## 📋 О проекте

**RentEase** - это современная веб-платформа для аренды недвижимости, автоматизирующая полный цикл сделки: от поиска объекта до подписания договора. Система обеспечивает удобное взаимодействие между арендаторами, собственниками и агентами, предоставляя интуитивно понятный интерфейс и мощный функционал.

### 🎯 Ключевые возможности

- **🔍 Поиск и фильтрация** - многофакторный поиск по цене, площади, типу недвижимости, количеству комнат; полнотекстовый поиск по адресу и названию ЖК
- **📝 Управление заявками** - полный жизненный цикл заявки: создание, обработка, одобрение/отклонение, автоматическое создание договора
- **📄 Документооборот** - автоматическая генерация договоров при одобрении заявки через триггеры PostgreSQL
- **💬 Коммуникации** - система сообщений в реальном времени с WebSocket и индикатором онлайн-статуса пользователей
- **👤 Личный кабинет** - управление профилем, аватаром, контактными данными и документами
- **📊 Аналитика** - статистика для агентов с визуализацией данных (Chart.js)
- **🏠 Управление объектами** - CRUD операции с недвижимостью, загрузка фотографий
- **🔐 Безопасность** - JWT-аутентификация, хеширование паролей, защита от SQL-инъекций

## 🏗 Архитектура

### Технологический стек

#### Backend
- **FastAPI** - высокопроизводительный асинхронный фреймворк
- **PostgreSQL 13+** - надежная реляционная СУБД
- **SQLAlchemy** - ORM для работы с базой данных
- **Pydantic** - валидация данных и управление схемами
- **JWT** - безопасная аутентификация через HttpOnly cookies
- **WebSockets** - real-time коммуникация и онлайн-статусы

#### Frontend
- **HTML5/CSS3** - адаптивная верстка с поддержкой мобильных устройств
- **JavaScript (ES6)** - интерактивность и динамическое обновление
- **Chart.js** - визуализация статистических данных
- **HTMX** - асинхронные запросы без перезагрузки страницы

## 📊 Модель данных

### Основные сущности

```sql
- users          # Пользователи (арендаторы, собственники, агенты, админы)
- properties     # Объекты недвижимости с характеристиками
- property_photos # Фотографии объектов (до 10 на объект)
- applications   # Заявки на аренду с историей статусов
- contracts      # Договоры аренды (авто-генерация при одобрении)
- messages       # Сообщения между пользователями (WebSocket)
- audit_logs     # Логи действий для аудита
```

## 🚀 Инструкция по запуску

### Требования

- **Python** 3.9 или выше
- **PostgreSQL** 13 или выше
- **pip** (менеджер пакетов Python)
- **Git** (для клонирования репозитория)

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/rentease.git
cd rentease
```

### 2. Создание виртуального окружения

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка базы данных

#### 4.1. Создание базы данных PostgreSQL

```sql
CREATE DATABASE "RentEase";
```

#### 4.2. Загрузить схему и данные из `database/RentEase.sql` туда через формат Plain в окне Restore

#### 4.3. Настройка подключения
Откройте файл `database.py` и измените строку подключения:
```python
POSTGRESQL_DATABASE_URL = "postgresql+psycopg2://postgres:1234@localhost:5432/RentEase"
```

### 5. Настройка подключения
Откройте файл `smtp.py` и настройте параметры:
```python
SMTP_SERVER = "smtp.yandex.ru"
SMTP_PORT = 587
SMTP_USER = "your-email@yandex.ru"
SMTP_PASSWORD = "your-app-password"
```

### 6. Настройка безопасности
Откройте файл `app.py` и измените секретный ключ:
```python
SECRET_KEY = "your-secret-key-here-change-in-production"  # ⚠️ Обязательно сменить!
```

### 7. Запуск приложения

```bash
python app.py
```
Или через uvicorn:
```bash
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

### 8. Проверка работы
- **Главная страница:** http://localhost:8000
- **API документация:** http://localhost:8000/docs

## ⚠️ Важные замечания
1. **Безопасность:** Обязательно измените `SECRET_KEY` перед развертыванием
2. **SMTP:** Используйте пароль приложения, а не пароль от аккаунта
3. **Файлы:** Папка `static/uploads/` должна иметь права на запись
4. **База данных:** Регулярно делайте бэкапы PostgreSQL
